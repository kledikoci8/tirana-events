🔴 CRITICAL ISSUES (MUST FIX IMMEDIATELY)
═══════════════════════════════════════════════════════════════

[CRITICAL-1] HARDCODED API URLs IN FRONTEND
Location: mobile/src/services/personalizationService.js:4
Issue: const API_URL = 'http://192.168.1.6:8080/api'
Impact: App will break in production, different environments
Fix Required:
  - Create environment configuration file
  - Use expo-constants for environment variables
  - Support dev/staging/production URLs

[CRITICAL-2] HARDCODED API URLs IN ALL SCREENS
Location: Multiple screens (OnboardingScreen.js:23, FilterScreen.js:48, 
          FriendsScreen.js:24, NotificationSettingsScreen.js:67, etc.)
Issue: Each screen has 'http://localhost:8080/api/...' hardcoded
Impact: Maintenance nightmare, inconsistent URLs, production failure
Fix Required:
  - Centralize API configuration
  - Import API_URL from config file
  - Replace all hardcoded URLs with config constant

[CRITICAL-3] NO JWT TOKEN REFRESH MECHANISM
Location: Frontend - No refresh token implementation
Backend: AuthController.java - Only login/register, no refresh endpoint
Issue: Tokens expire (jwt.expiration) with no way to refresh
Impact: Users forced to re-login frequently, poor UX
Fix Required:
  - Add /api/auth/refresh endpoint in backend
  - Implement refresh token storage (secure)
  - Add axios interceptor to auto-refresh on 401
  - Handle token expiration gracefully

[CRITICAL-4] NO 401/403 GLOBAL ERROR HANDLING
Location: Frontend - No axios interceptors configured
Issue: Each API call handles errors individually, no global auth error handling
Impact: Inconsistent behavior on token expiration, no auto-logout
Fix Required:
  - Add axios response interceptor
  - Detect 401/403 responses globally
  - Auto-logout and redirect to login
  - Clear AsyncStorage on auth failure

[CRITICAL-5] MISSING SERVER-SIDE VALIDATION
Location: Multiple controllers
Issue: Controllers rely on @Valid but don't validate business logic
Examples:
  - FilterController: No validation that maxDistance > 0
  - SocialController: No check if user can unfollow themselves
  - TicketController: No validation of ticket ownership before transfer
Impact: Data corruption, security vulnerabilities, business logic bypass
Fix Required:
  - Add business logic validation in service layer
  - Validate ownership before operations
  - Check constraints (distance > 0, dates valid, etc.)
  - Return proper error messages

[CRITICAL-6] NO RATE LIMITING
Location: Backend - No rate limiting configured
Issue: All endpoints are unprotected from abuse
Impact: DDoS vulnerability, spam, resource exhaustion
Fix Required:
  - Add Spring rate limiting (Bucket4j)
  - Limit auth endpoints (5 attempts/minute)
  - Limit notification endpoints (10/minute)
  - Limit chat endpoints (20/minute)

[CRITICAL-7] SENSITIVE DATA IN ERROR RESPONSES
Location: Multiple controllers using RuntimeException
Issue: throw new RuntimeException("User not found") exposes internal state
Impact: Information leakage, security vulnerability
Fix Required:
  - Create custom exception classes
  - Use @ControllerAdvice for global exception handling
  - Return generic error messages to client
  - Log detailed errors server-side only

[CRITICAL-8] NO INPUT SANITIZATION
Location: Frontend - User inputs not sanitized before API calls
Backend: No XSS protection on text fields
Issue: Chat messages, event descriptions, reviews not sanitized
Impact: XSS attacks, SQL injection (if raw queries used), data corruption
Fix Required:
  - Frontend: Sanitize inputs before sending
  - Backend: Add @JsonIgnore for sensitive fields
  - Backend: Use OWASP Java HTML Sanitizer
  - Validate/escape HTML in descriptions

[HIGH-4] MISSING NETWORK ERROR HANDLING

Impact: App crashes or hangs when network is poor

[HIGH-5] NO OPTIMISTIC UPDATES

[MEDIUM-4] NO DEBOUNCING ON SEARCH/FILTER

Impact: Excessive API calls, poor performance, high server load

[MEDIUM-5] MISSING ACCESSIBILITY LABELS
