FILE: backend/src/main/java/com/tirana/events/service/TicketService.java

PROBLEM:
- Stock was decremented AFTER ticket was saved
- Two users could buy the last ticket simultaneously
- Risk of overselling tickets

SOLUTION:
- Stock is now decremented BEFORE creating ticket
- Added atomic check: if stock <= 0, throw exception immediately
- Pessimistic lock already in place (findByIdForUpdate)
- Transaction ensures atomicity

CHANGES:
1. Moved stock decrement from AFTER ticket save to BEFORE
2. Added explicit check: if ticketsAvailable <= 0, throw ConflictException
3. Added comments explaining the fix

RESULT:
✅ No more overselling
✅ Race condition eliminated
✅ Data integrity maintained
